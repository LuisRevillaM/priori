## What Changed

SCP-0 adds a semantic registry layer over the current executable system. It does not change runtime behavior, Hermes behavior, Workbench UI behavior, data loading, tactical primitives, or recipe execution.

Commit under review:

`b348595 Add SCP-0 semantic registry foundation`

Implemented:

- Versioned Pydantic schemas for concepts, operationalizations, definitions, runtime bindings, operator definitions, recipes, composition instances, evidence contracts, claim contracts, exposure policies, maturity records, and projection policies.
- Runtime manifest generation from the current executable catalog and checked-in query plans.
- Fail-closed registry validation against runtime truth.
- Registry lock generation with semantic and runtime hashes.
- Product, AI, recipe-library, unsupported-capability, and research-atlas projections.
- Current-system mappings for all active runtime primitive/relation capabilities, all current operators, four registered recipes, and one validated AI-authored composition.
- Import of the 741-entry five-year capability atlas as `PROPOSED_ATLAS`, denied from product and AI projections.
- A parity report for two pilot traversals: High-Bypass Completed Pass and Ball-Side Block Shift.
