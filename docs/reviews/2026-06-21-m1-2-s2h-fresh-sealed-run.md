- adds bounded aliases for existing approved-recipe and corridor families;
- adds support-clarification aliases for the new ambiguous phrases;
- expands unsupported capability-code aliases;
- derives host-owned gap codes from both model refusal text and the original
  request, preventing safe refusals from being under-credited when the model
  summarizes an unsupported concept.

No runtime semantics, tactical primitives, recipe families, replay behavior, or
UI changed.
