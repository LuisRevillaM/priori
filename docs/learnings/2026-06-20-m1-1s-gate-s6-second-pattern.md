# Learning: The Second Pattern Should Start From A Different Anchor Domain

The strongest way to test the generic runtime after S4 was not to add a new tactical primitive. It was to reuse the existing corridor relation from a different anchor domain.

The possession-corridor plan starts from ordinary possession anchors and asks whether a progressive corridor appears. That is enough to prove the runtime can emit real rows without the M1 wide-entry, defensive-shift, outcome, or destination-entry spine.

Two lessons matter for future agents:

- Relations should accept semantic anchor records directly instead of requiring result-shaped records.
- A second-pattern proof should keep the plan small so failures identify architecture coupling, not feature sprawl.
